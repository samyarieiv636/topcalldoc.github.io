CALL功能介绍

CALL功能分为自动call和手动call两种，也称自动派彩和手动派彩，自动call主要为了解决玩家的游戏体验感，手动call主要是把这个调用的方式交由给运营商来定制化使用

CALL Feature Introduction

The CALL feature is divided into two types: automatic call and manual call, also known as automatic payout and manual payout. Automatic call is mainly to solve the player's gaming experience, while manual call is mainly to hand over the calling method to operators for customized use.

请注意，call功能默认不开放后台配置权限，确认后可以向我方技术申请开启，自动call和手动call功能权限是放在一起的，需要知悉以下风险：

手动call影响的点可能有

  1. 除正常的GGR收费以外，手动call出去的金额会抵消掉系统正常派彩部分，乘以费率进行交收。

    例:

    正常收费的情况下：

    (下注-系统正常派彩)*费率

    玩家下注100，系统正常派彩90，那收费是=10*费率

    手动call的情况下：系统正常派彩=0

    (下注-(系统正常派彩+call派彩)+call派彩)*费率

    玩家下注100，call派彩200，那收费是=100*费率

  2. 还有一个点影响是从贵司角度，比如贵司后台账号并未管理到位，有可能会有内部人员未经允许使用来为指定玩家派彩的情况，会有可能有造成损失

  3. 如果有使用call功能的需求，建议只使用自动Call，后台设置自动Call没有收费。

如申请开通即认为贵司已经知晓以上风险，使用手动call不当导致产生的费用都是需要交收的，请你理解

Please note that the call feature is not open for backend configuration permissions by default. After confirmation, you can apply to our technical team to enable it. The permissions for automatic call and manual call are placed together. Please be aware of the following risks:

The impact points of manual call may include:

  1. In addition to normal GGR charges, the amount manually called out will offset the system's normal payout portion and be settled by multiplying the rate.

    Example:

    Under normal charging conditions:

    (Bet - System Normal Payout) * Rate

    Player bets 100, system normal payout 90, then the charge = 10 * Rate

    Under manual call conditions: System normal payout = 0

    (Bet - (System Normal Payout + Call Payout) + Call Payout) * Rate

    Player bets 100, call payout 200, then the charge = 100 * Rate

  2. Another impact point is from your company's perspective. For example, if your company's backend accounts are not properly managed, there may be situations where internal personnel use it without authorization to pay out to designated players, which may cause losses.

  3. If there is a need to use the call feature, it is recommended to only use automatic Call. There is no charge for setting automatic Call in the backend.

If you apply for activation, it is considered that your company is aware of the above risks. Fees incurred from improper use of manual call are subject to settlement. Please understand.

自动call（自动派彩）

请留意：自动call算法生效需要大量的下注数据，当前目标RTP计算的值和实际RTP计算的值误差大于2000本代理币种的时候是最容易触发

比如：

目标RTP=90的时候，当前总下注10000，总派彩9000，那不容易触发自动call算法

目标RTP=90的时候，当前总下注10000，总派彩6000，那容易触发自动call算法

  1. 菜单入口: Slot>Auto Call Setting

  2. 选择对应的游戏供应商

  3. 配置页面介绍

    1. Percent：整体扣除RTP的数值，比如：5，那整体玩家的目标RTP都-5，扣除的这5部分的RTP，会用来随机给少部分玩家派彩，以达到有小部分玩家真正盈利的情况，但是整体agent的数据还是会保持盈利的状态，建议配置：5~10范围

    2. MaxAmount Per Day：每个玩家每天能获取到的最大派彩金额，0是没有限制，默认是0

    3. Trigger Balance：触发的余额配置，比如配置是 100，那只有可能玩家余额大于100的时候才有可能触发，默认是0，不限制余额

    4. Trigger Total Bet：触发的下注额配置，比如配置是 100，那只有可能玩家下注大于100的时候才有可能触发，默认是0，不限制余额

    5. Mode: Enable开启，Disable关闭

Automatic Call (Automatic Payout)

Please note: The automatic call algorithm requires a large amount of betting data to take effect. It is most likely to trigger when the error between the current target RTP calculated value and the actual RTP calculated value is greater than 2000 units of the agent currency.

For example:

When target RTP = 90, current total bet 10000, total payout 9000, it is not easy to trigger the automatic call algorithm

When target RTP = 90, current total bet 10000, total payout 6000, it is easy to trigger the automatic call algorithm

  1. Menu entry: Slot>Auto Call Setting

  2. Select the corresponding game provider

  3. Configuration page introduction

    1. Percent: The value of overall RTP deduction, for example: 5, then all players' target RTP will be -5. The deducted 5% RTP will be used to randomly pay out to a small number of players to achieve a situation where a small number of players actually profit, but the overall agent data will still maintain a profitable state. Recommended configuration: 5~10 range

    2. MaxAmount Per Day: The maximum payout amount each player can receive per day. 0 means unlimited, default is 0

    3. Trigger Balance: Trigger balance configuration, for example, if configured as 100, it will only trigger when the player's balance is greater than 100. Default is 0, no balance restriction

    4. Trigger Total Bet: Trigger bet amount configuration, for example, if configured as 100, it will only trigger when the player's bet is greater than 100. Default is 0, no bet restriction

    5. Mode: Enable to turn on, Disable to turn off

手动call（手动派彩）

需要留意：手动call会产生额外的消耗费用，详细请参考以上功能介绍中的详细介绍

  1. 菜单入口: Slot>Active User & Call

  2. 选择对应的游戏供应商

  3. 配置页面介绍

    1. Online users：显示当前在线玩家，只能派送给当前在线玩家

    2. 在Controll 列中，有小礼物标识，点击打开派送选项

    3. 例如下拉选择中：1320.45x （Free）> 26,409

      1. 1320.45：表示赠送当前下注额的倍数

      2. Free：表示赠送免费旋转

      3. JackPot：表示赠送奖池派彩

      4. 26,409：表示大概赠送的总金额（此部分是大概金额，并不是完全精确，请留意）

    4. 点击确认配置，等待系统给玩家派送奖励

手动call配置之后不一定会派送成功，如果玩家退出游戏，更换游戏，修改下注金额可能是会导致手动派彩的失败，具体是否派送成功，请参考派送的历史记录进行确认，查看派送历史记录步骤：菜单Game History>Call log

  • 等待:调用已经发出,但用户尚未进行旋转(在这种情况下,您可单击按钮取消调用)。
  • 处理中:正在应用投注(即用户正在赢取投注金额)
  • 应用:已应用（派送成功）

请留意：手动派彩会影响到整体RTP的计算，比如正常整体RTP维持在95，因手动派彩导致超过100的情况我方不负责，辛苦留意

Manual Call (Manual Payout)

Please note: Manual call will generate additional consumption fees. For details, please refer to the detailed introduction in the feature introduction above.

  1. Menu entry: Slot>Active User & Call

  2. Select the corresponding game provider

  3. Configuration page introduction

    1. Online users: Display current online players, can only be sent to current online players

    2. In the Control column, there is a small gift icon. Click to open the delivery options

    3. For example, in the dropdown selection: 1320.45x (Free) > 26,409

      1. 1320.45: Indicates the multiple of the current bet amount to be given

      2. Free: Indicates free spins to be given

      3. JackPot: Indicates jackpot payout to be given

      4. 26,409: Indicates the approximate total amount to be given (This is an approximate amount, not completely accurate, please note)

    4. Click to confirm the configuration and wait for the system to send rewards to players

Manual call configuration may not necessarily be successful. If the player exits the game, changes games, or modifies the bet amount, it may cause manual payout failure. To confirm whether the payout was successful, please refer to the payout history. Steps to view payout history: Menu Game History>Call log

  • Waiting: The call has been sent, but the user has not yet spun (in this case, you can click the button to cancel the call).
  • Processing: Applying the bet (i.e., the user is winning the bet amount)
  • Applied: Applied (payout successful)

Please note: Manual payout will affect the overall RTP calculation. For example, if the normal overall RTP is maintained at 95, and manual payout causes it to exceed 100, we are not responsible. Please note.
